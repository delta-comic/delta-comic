## app侧清单

- [x] 现在应用所使用的日志库过于原始且不统一，请你重构应用的日志系统，确保前端和rust共用一套体系，以下是具体要求
  - 创建一个新的子repo，名叫`@delta-comic/logger`
  - 前端和rust都要有接入，所以可能要用到tauri-plugin
  - rust库使用`tracing`+`tracing-subscriber`
  - 要有日志的即时写入，自动分块，规则你决定
  - 控制台输出也要有
  - 定期的日志压缩归档，过于久远的日志自动删除。规则为1月一归档，3月一删除
  - 一定要使用非阻塞异步io
  - 涵盖常见的日志分级，生产环境最低为`info`
  - 支持多分组日志与格式化，我的预期格式为：`[yyyy/mm/dd hh:MM:ss] (scope) level > content`
  - 充分利用oop的思想和class/impl去设计规划代码，不要有太多的free-function
  - 代理前端应用的console(复制一份)，并捕获前端和rust的错误一并写入日志
  - 保证性能优良
  - 在网页端环境应自动回退到console实现
- [x] 当你完成了编写时，你还要做的是
  - [x] 为全仓的代码加入log。关键的节点和便于debug的位置与持久性任务的进度都要有log。
  - [x] 提供一个应用内置的日志阅读器和导出功能，放在设置里，注意响应式设计，在网页端不用有
  - 该子repo应当对外发布以供插件下载调用
